# Caddy

Caddy وظیفهٔ امن‌سازی و ارائهٔ HLS از طریق TLS را دارد. این‌جا باید دقت کنیم که مسیرها دقیقاً با MediaMTX هم‌خوان باشند.

## وظایف اصلی

- Terminate کردن TLS
- reverse proxy یا file serve به مسیر HLS
- تنظیم پورت‌های خارجی

## نکات حیاتی

- اگر Caddy روی پورت خاصی (مثلاً 8443) گوش می‌دهد، کلاینت باید دقیقاً همان پورت را بزند.
- اگر روی 443 قرار گرفته باشد، دیگر نیازی به پورت در URL نیست.

## نمونه ساختار (مفهومی)

```
cam.hydroleaf.se:8443 {
  reverse_proxy localhost:8888
}
```

> در این مثال، HLS از MediaMTX روی 8888 آمده و Caddy آن را به بیرون ارائه می‌دهد.
